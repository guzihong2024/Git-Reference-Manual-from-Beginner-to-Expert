1. 基础命令 (Command Line & Git Basics)

命令行操作 (Terminal / PowerShell)

  • cd ~/Desktop/folder: 更改命令行当前运行的文件夹。
  • ls (或 dir): 列出当前文件夹中的文件和文件夹。
注意： git 命令必须在包含所有代码的文件夹内运行。

配置提交信息

配置用于提交的姓名和邮箱：

  • git config --global user.name "Your Name"
  • git config --global user.email "email@example.com"

创建提交 (Creating Commits)

在 Git 中，**版本 (version) = 提交 (commit)**。
**版本历史 (Version history) = 提交历史 (commit history)**。

git init Git 将开始追踪所有更改。
git status 显示自上次提交以来的所有更改。
git add <file|folder> 挑选更改以放入下一次提交中。
git add . 挑选所有文件 (在当前运行文件夹)。
git commit -m "msg" 创建一个附带消息的提交。
git commit -m "msg" --amend 更新上一次提交而不是创建新的。
git log 查看提交历史。
git log --all --graph 在命令行中以可视化的方式显示分支结构。

2. 工作区与历史 (Working Area & History)

工作区 vs 暂存区

Working Area (工作区)
⬇︎ git add <file|folder>
Staging Area (暂存区)
⬇︎ git commit -m "..."
Commit History (提交历史)
git add <file> 工作区 => 暂存区
git commit -m "..." 暂存区 => 提交历史
git reset <file> 暂存区 => 工作区 (撤销暂存)
git checkout <file> 工作区 => 移除更改 (丢弃修改)

查看与恢复文件 (Restoring)

HEAD 指向当前工作的分支或提交。

git checkout <hash|branch> 切换到指定的提交或分支。
git checkout <hash> <file> 将单个文件内容恢复到之前的提交状态。
git checkout <hash> folder/ 恢复文件夹内的所有文件。
git checkout <hash> . 恢复项目中的所有文件。

其他功能

  • 别名: git config --global alias.s "status"
  • .gitignore: 告诉 git 哪些文件/文件夹不应该被追踪。
  • 移除 Git: rm -rf .git (从项目中移除 git 追踪)。

3. GitHub 与远程仓库 (GitHub & Remotes)

上传代码 (Uploading)

远程仓库 (Remote) 用于线上备份代码。

git remote add <name> <url> 链接本地仓库到远程仓库 (如 git remote add origin ...)。
git push <remote> <branch> 将分支上传到远程仓库 (如 git push origin main)。
git push -u origin main 设置上游快捷方式，下次只需 git push。
git push origin main -f 强制推送 (会覆盖远程仓库内容)。

下载与同步 (Downloading & Syncing)

git clone <url> 下载远程仓库。
git remote -v 列出所有远程仓库的详细信息。
git fetch 更新所有远程追踪分支，查看远程仓库的样子。
git pull <remote> <branch> 下载远程分支的新提交，并更新本地分支 (Fetch + Merge)。

4. 分支与合并 (Branching & Merging)

分支 (Branching)

分支 = 创建版本历史的副本，以便并行开发新功能。

git branch <name> 创建一个新分支。
git checkout <name> 切换到另一个分支。
git branch -D <name> 删除一个分支。

合并 (Merging)

命令：git merge <branch_name> -m "message"

步骤：1. git checkout main (切换到目标分支)。2. git merge feature1 (合并另一个分支)。

解决合并冲突 (Merge Conflicts)

冲突标记示例：

<<<<<<< HEAD
code1 (当前分支的代码)
=======
code2 (合并进来的分支的代码)
>>>>>>> branch

解决步骤：

  1. 删除所有标记符号，只保留想要的最终代码。
  2. 提交更改：git add . 然后 git commit -m "message"。

5. 功能分支工作流 (Feature Branch Workflow)

公司开发新功能的常用流程：

工作流步骤

  1. **创建分支：** git branch new-feature -> git checkout new-feature
  2. **开发与提交：** 修改代码并提交。
  3. **上传：** git push origin new-feature
  4. **创建 PR：** 在 GitHub 网页上发起 Pull Request (用于代码审查)。
  5. **合并：** 在 GitHub 上点击 "Merge pull request"。
  6. **更新本地：** 同步本地 main 分支： 
    git checkout main
git pull origin main

工作流中的冲突解决

方法 1: 在 GitHub 网页上解决

⚠︎ This branch has conflicts that must be resolved
Use the web editor or the command line to resolve conflicts.

方法 2: 在本地电脑上解决 (推荐)

  1. **获取 main：** git checkout main -> git pull origin main
  2. **获取 feature：** git checkout feature4 -> git pull origin feature4
  3. **将 main 合并进 feature：** 
    git checkout feature4
git merge master
    (注意合并方向：将 merge commit 保留在 feature 分支上供队友审查)
  4. **推送：** 解决冲突后，git push origin feature4。
